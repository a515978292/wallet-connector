# ETH 和 ERC20 余额区别

> 理解以太坊中的两种余额类型

---

## 两种余额的本质区别

### 1️⃣ **ETH（以太币）余额**

- **原生代币**：ETH 是以太坊区块链的原生货币
- **存储位置**：直接存储在账户的状态中，由区块链底层协议管理
- **用途**：
  - 支付交易的 Gas 费用
  - 作为价值转移的媒介
  - 用于智能合约交互的手续费
- **查询方式**：通过区块链节点直接查询地址余额
- **转账方式**：通过普通的以太坊转账交易

### 2️⃣ **ERC20 代币余额**

- **智能合约代币**：ERC20 是基于智能合约实现的代币标准
- **存储位置**：存储在特定智能合约的内部映射（mapping）中
- **常见例子**：
  - USDT、USDC（稳定币）
  - UNI（Uniswap 代币）
  - LINK（Chainlink 代币）
- **查询方式**：调用合约的 `balanceOf(address)` 函数
- **转账方式**：调用合约的 `transfer()` 或 `transferFrom()` 函数

---

## 🏦 形象比喻

- **ETH** 就像你银行账户里的**现金余额**，由银行（区块链）直接管理
- **ERC20** 就像你持有的**股票、基金或积分**，由各个公司（智能合约）分别记录

---

## 在 useBalance 中的使用

```typescript
// 查询 ETH 余额
const { data: ethBalance } = useBalance({
  address: "0x...",
});

// 查询特定 ERC20 代币余额（如 USDT）
const { data: usdtBalance } = useBalance({
  address: "0x...",
  token: "0xdac17f958d2ee523a2206206994597c13d831ec7", // USDT 合约地址
});
```

**关键差异**：

- 查询 ETH 时：**不需要**指定 `token` 参数
- 查询 ERC20 时：**必须**提供代币合约地址

---

## 🔑 核心要点

1. **一个钱包地址可以同时持有**：

   - 1 种 ETH 余额
   - N 种 ERC20 代币余额

2. **为什么支持两种查询**：

   - ETH 是区块链原生支持的，直接查询
   - ERC20 是智能合约，需要调用合约函数

3. **Gas 费用**：

   - 所有交易的 Gas 费都必须用 ETH 支付
   - 即使只转账 ERC20 代币，也需要 ETH 支付 Gas

4. **安全提示**：
   - 使用 ERC20 时务必确认合约地址正确
   - 不同链上同一代币的合约地址不同
